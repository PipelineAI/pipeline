REST API documentation including POST and GET syntax.

Predict


              curl -X POST -H "Content-Type: application/json" -d '{requestBody}' \
                https://prediction-python3.pipeline.io:80/api/v1/model/predict/{model_type}/{model_namespace}/{model_name}/{model_version}

Key Parameters

The following key parameters are used to establish model unique id's and api endpoints.
  • api endpoint = {model_type}/{model_namespace}/{model_name}/{model_version}
  • model unique id = {model_type}_{model_namespace}_{model_name}_{model_version}
model_type ml/ai model type, no underscores; Examples: jvm; python3; tensorflow; scikit
model_namespace arbitrary namespace typically used to segment models by topic, "default" by default, no underscores; Examples: census-ml; vacation-rental-property-ml
model_name usually the logical name of your model including the type of logic or library used, no underscores; Examples: sklearn; tensorflow-cpu; tensorflow-gpu; theano
model_version version of model; unique within {model_namespace}/{model_name}; ie. v0, myModelA, yourModel_9

Command Line Interface (CLI)

Upload new model versions and make predictions from the bash prompt.

PipelineIO Command Line Interface (CLI)


              pip install pio-cli

Upload New Model Version


              pio model_deploy --model_server_url={model_server_url}:81 --model_type={model_type} \
               --model_namespace={model_namespace} --model_name={model_name} --model_version={model_version} \
              --model_bundle_path=/path/to/{model_type}/{model_namespace}/{model_name}/{model_version}/

Predict


              pio model_predict --model_server_url={model_server_url} --model_type={model_type} \
              --model_namespace={model_namespace} --model_name={model_name} --model_version={model_version} \
              --input_file_path=/path/to/inputs.json
